事务

1.首先与客户端进行连接(TCP)

网络不通

2.建立连接，客户端发送请求给服务器(http)

3.服务端接受请求，给与响应信息(HTTP)

4.客户端接受服务器返回的信息用过浏览器显示在用户显示屏上，断开连接

请求与响应

请求行组成

响应报文

req accept 接受 accept-encoding 接不接受压缩 connection：keep-alive 长连接 host 访问网站的域名 user-agent 浏览器向服务器介绍自己 res remote dns解析后得到的IP

状态码

1XX 异步处理 后端通信与后端通信 2XX 成功处理 3XX 重定向(服务器) 本地重定向(本地缓存) 4XX 浏览器或者客户端操作有问题 5XX 服务端错误 服务端未能实现合法请求

head只返回头部信息

cookie和session

http无状态协议 session 一般在内存 小数据库 cookie sessionID 服务器发送的session token(手机app) 可以嵌到url 只使用一次就作废 安全性更高

缓存

优点 减少请求 减少带宽消耗 再次请求 没过期 缓存 过期 ETAG策略 IF-none-match

链路安全

链路加密 https 对重要数据加密 http1 http2 http2 传输二进制格式 高效 紧凑 抱头压缩 降低开销 多路复用 网络 服务器主动推送 ws不是基于http
